3.18
发现角色下落过程中左右移动时，若触碰到踏板会导致角色与踏板发生碰撞
本想通过关闭碰撞器来解决问题，但发现如果关闭碰撞器，角色起跳时速度不为0
角色跳跃高度波动较大
3.22
修改了跳板生成机制，通过确定角色位置来确定跳板生成的位置
因为电脑屏幕较手机屏幕宽很多，若满屏生成跳板较为不合理。
3.23
修改了跳板生成的条件，原来的条件是当相机上移时生成随机数量的跳板
但这样做有较多缺陷：1、因为相机位置与角色有关，虽然角色跳跃高度波动不大
但仍然使跳板生成条件被触发，又因为跳板生成条件的检测是每一帧都进行的，故这样会导致跳板鬼畜生成......
2、相机位置是浮点数，浮点数比较存在一些不确定因素
故舍弃原有的触发条件。
新的生成触发机制由于对原游戏的观察发现，除了部分有怪物站的跳板外，同一高度似乎没有两块跳板（也不清楚有没有看错）
故新的机制为每次生成高度较上一次一定，但有不生成的概率
